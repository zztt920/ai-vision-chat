# AI 视讯助手 — 设计文档

> 版本: 1.0 | 日期: 2026-06-12 | 状态: 设计完成

## 1. 概述

### 1.1 产品定位

一款基于 Web 浏览器的 AI 视觉对话应用，通过摄像头与麦克风实现多模态交互——AI 能"看到"用户摄像头画面中的内容，"听到"用户的语音提问，并给予恰当的语音+文字回复。

### 1.2 核心目标

- 演示摄像头视觉理解 + 语音交互的端到端能力
- 实现零 API 运营成本（利用免费资源）
- 提供流畅自然的对话体验

### 1.3 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | HTML5 + CSS3 + Vanilla JS | 单页应用，无框架依赖 |
| 摄像头 | WebRTC (getUserMedia) | 浏览器原生 API |
| 语音识别 | Web Speech API (SpeechRecognition) | 浏览器原生 STT，免费 |
| 语音合成 | Web Speech API (SpeechSynthesis) | 浏览器原生 TTS，免费 |
| 视觉理解 | Google Gemini 1.5 Flash API | 免费层 60 RPM |
| 后端代理 | Node.js (Express) | 轻量级 API 代理 |
| 部署 | Vercel / GitHub Pages | 免费托管 |

### 2.1 数据流

1. **用户语音输入**：浏览器 SpeechRecognition API 持续监听，识别结果转为文本
2. **摄像头画面采集**：getUserMedia 获取视频流，Canvas 以 1-2 fps 截取帧，检测场景变化
3. **请求发送**：文本 + 图像帧（JPEG 压缩）通过 Node.js 代理转发至 Gemini API
4. **AI 处理**：Gemini 1.5 Flash 分析图像内容与文本提问，生成回复文本
5. **回复呈现**：回复文本显示在对话区，同时通过 SpeechSynthesis API 朗读
6. **持续对话**：上下文历史维护，支持多轮对话

### 2.2 核心模块

#### 前端模块
- **CameraModule**：摄像头初始化、帧捕获、场景变化检测、帧率控制
- **SpeechModule**：语音识别（STT）与语音合成（TTS）管理
- **ChatModule**：对话消息管理、消息渲染、上下文维护
- **APIClient**：与 Node.js 代理层的通信
- **UI Controller**：状态指示、控件交互、界面更新

#### 后端模块
- **Proxy Server**：API Key 保护、请求转发、CORS 处理
- **Rate Limiter**：请求频率控制，避免超限
- **Logger**：请求日志记录，便于调试与监控

## 4. 运营成本控制

### 4.1 成本控制策略汇总

| 策略 | 描述 | 状态 |
|------|------|------|
| 浏览器原生 STT | Web Speech API 语音识别，零费用 | ✅ 已实现 |
| 浏览器原生 TTS | Speech Synthesis API 语音合成，零费用 | ✅ 已实现 |
| Gemini 免费层 | 60 RPM 免费额度覆盖演示需求 | ✅ 已实现 |
| 智能帧率控制 | 1-2 fps 采集，场景无变化不发送（`camera.js` 像素级对比） | ✅ 已实现 |
| 图像压缩 | JPEG 质量 60%，分辨率降至 320px（`utils.js` compressImage） | ✅ 已实现 |
| 上下文压缩 | 超过 20 条对话自动摘要合并（`chat.js` compressHistory） | ✅ 已实现 |
| 空闲暂停 | 用户 15 秒无活动自动暂停帧采集（`app.js` idleTimer） | ✅ 已实现 |

### 4.2 实际采用的成本控制分析

所有 7 项策略均在本版本中实现：

| 想法来源 | 是否采用 | 实现方式 |
|---------|---------|---------|
| 用浏览器原生 API 替代付费 STT/TTS | ✅ 采用 | `speech.js` 调用 Web Speech API |
| 用 Gemini 免费层替代付费模型 | ✅ 采用 | 后端 `server/index.js` 指向 Gemini 1.5 Flash 免费端点 |
| 降低帧率减少 API 调用量 | ✅ 采用 | `camera.js` 固定 500ms 间隔（2 fps） |
| 场景变化检测避免发送无变化的帧 | ✅ 采用 | `utils.js` 像素级 diff，阈值 5% |
| 降低图片分辨率/质量减 payload | ✅ 采用 | `compressImage` 缩放到 320px，JPEG 60% |
| 长对话自动摘要减少 tokens | ✅ 采用 | `chat.js` 超 20 条自动合并前 10 条 |
| 空闲时停采避免无效传输 | ✅ 采用 | `app.js` 15 秒无活动空闲计时器 |

## 5. 界面设计

采用 **左屏右聊** 双栏布局：

- **左侧（60%）**：摄像头实时画面预览，底部工具栏（暂停/切换摄像头）
- **右侧（40%）**：对话消息列表（语音气泡 + 文字），顶部状态指示器
- **底部**：当前状态栏，显示 AI 等待/处理/回复状态

状态指示系统：
- 顶部状态栏：摄像头连接、麦克风状态、AI 连接状态
- 侧边指示器：STT 就绪、TTS 就绪
- 底部状态栏：当前 AI 行为状态

## 6. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 摄像头权限拒绝 | 显示引导提示，支持重试 |
| 麦克风权限拒绝 | 降级为纯文字输入模式 |
| Gemini API 超限 | 显示"请稍后再试"，自动退避重试 |
| 网络断开 | 检测到断开时显示提示，自动重连 |
| 浏览器不支持 Web Speech | 降级为文字输入 + 文字输出 |
